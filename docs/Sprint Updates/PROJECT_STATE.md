# Current Development Philosophy

Discovery has entered a new development phase.

The reasoning engine is no longer the primary bottleneck.

The Executive Experience is.

Before implementing any new reasoning capability, always determine:

- Does Discovery already produce this capability?
- Is it already persisted in runtime?
- Is it already benchmarked?
- Is it already projected through the Executive Projection?
- Is it already visible to the executive?
- Which executive question does it answer?
- Should it become a first-class executive concept?

Prefer exposing, validating, benchmarking, integrating, and pressure-testing existing organizational cognition before expanding Discovery's cognitive architecture.

Executive Projection is the canonical boundary between organizational cognition and executive communication.

React components should consume only the Executive Projection rather than reconstructing executive meaning directly from engine or runtime objects.
