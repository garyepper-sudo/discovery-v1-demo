# Discovery Architecture

This directory contains Discovery's canonical architecture, validation guidance, reference material, archives, and machine registries.

## Canon

Defines Discovery's permanent architecture, including:

- Cognitive Object Model
- Cognitive Flow Map
- Executive Cognitive Operating System
- Decision Intelligence Architecture
- Organizational Mechanism Lifecycle

Start with `Canon/COGNITIVE_ARCHITECTURE_INDEX.md` to locate the document that owns each architectural question.

`Canon/ORGANIZATIONAL_MECHANISM_LIFECYCLE.md` defines how mechanism identity persists, when understanding is revised rather than treated as new, lifecycle semantics, and longitudinal invariants. Only immediate-previous identity reconciliation is currently implemented; full lifecycle history remains deferred.

## Cognitive Operating Systems

Defines each major Cognitive Operating System and its responsibilities.

## Validation

Confirms actual implementation behavior against intended architecture.

Key documents include:

- Runtime Validation
- Executive Projection Audit
- Atlas Evaluation Loop

Architecture documents describe intended behavior. Runtime and benchmarks establish actual behavior.

## Reference

Contains supporting architecture documents and domain references.

## Archive

Contains historical architecture documents retained for context. Archived documents do not override the Canon or current Sprint Updates.

## Machine registries

The JSON files in this directory are generated architecture artifacts. Do not manually edit:

- `COGNITIVE_CAPABILITY_AUDIT.json`
- `COGNITIVE_CAPABILITY_REGISTRY.json`
- `COGNITIVE_FILE_REGISTRY.json`
- `DISCOVERY_ARCHITECTURE_STATE.json`
