# Discovery Canonical Architecture

## Philosophy

Discovery is not a collection of AI features.

Discovery is a cognitive architecture.

Every folder should represent one stage of cognition.

Every file should have one responsibility.

Every concept should have one canonical home.

# Major Engine Domains

model/
Owns persistent organizational knowledge.

semantic/
Extracts meaning from evidence.

cognition/
Transforms observations into understanding.

understanding/
Creates stable organizational understanding.

runtime/
Maintains longitudinal organizational state.

executive/
Produces executive communication.

expression/
Projects understanding into user interfaces.

---

# Living Understanding

Living Understanding is not the reasoning engine.

Living Understanding is the visual expression of organizational understanding.

It is derived from:

Organizational Understanding State

and rendered as:

Executive Organism

Explorer

Pattern Network

Health Metrics

# Architecture Rule

Never create duplicate implementations.

If two files perform the same responsibility:

Choose one canonical implementation.

Archive or remove the other.

---

# Refactoring Rule

Never move more than one subsystem at a time.

Every migration must:

compile

typecheck

deploy

before continuing.
