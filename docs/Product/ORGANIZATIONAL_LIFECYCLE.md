# Organizational Lifecycle

**Status:** Canonical

---

# Purpose

This document defines the canonical lifecycle of an organization inside Discovery.

It answers one question:

> **How does an organization continuously improve over time?**

Discovery is not a collection of features.

Discovery's current Executive Operating System application supports one continuously evolving organizational lifecycle.

Every Runtime object, workflow, API, interface, and reasoning capability exists to move an organization through this lifecycle.

If another product document conflicts with this one, this document is correct.

---

# Core Principle

Organizations continuously change.

Discovery continuously improves its understanding of those organizations.

Executives continuously improve the organization through Executive Work.

Every completed cycle should improve future executive judgment.

Within this application lifecycle, the Organization Model is the primary product representation.

---

# Canonical Lifecycle

```text
Organization

↓

Operating Model

↓

Continuous Understanding

↓

Executive Work

↓

Decision Support

↓

Executive Commitment

↓

Execution

↓

Review

↓

Learning

↓

Operating Model Evolution

↓

Repeat Forever
```

Discovery is never "finished."

The Operating Model should become progressively more accurate as organizations evolve.

---

# Stage 1 — Organization

Purpose

Represent a real organization.

Discovery initializes:

- Organization Runtime
- Organization Memory
- Operating Model

The Organization becomes the permanent unit of understanding.

---

# Stage 2 — Operating Model

Purpose

Maintain Discovery's current understanding of organizational reality.

The Operating Model continuously synthesizes:

- evidence
- organizational conditions
- organizational state
- executive assessment

The Operating Model is Discovery's current best explanation of how the organization works.

---

# Stage 3 — Continuous Understanding

Purpose

Continuously improve the Operating Model.

New evidence may include:

- uploaded documents
- executive observations
- organizational events
- decisions
- initiatives
- reviews
- outcomes

Discovery should improve understanding continuously rather than through isolated analyses.

---

# Stage 4 — Executive Work

Purpose

Represent organizational change.

Executive Work is the primary object managed by executives inside Discovery.

Executive Work includes:

- Executive Decisions
- Executive Initiatives

Executive Work may originate from:

- executives
- Discovery recommendations

Every Executive Work item progresses through the same lifecycle regardless of origin.

---

# Stage 5 — Decision Support

Purpose

Improve executive judgment before organizational commitment.

Discovery may expose:

- understanding
- constraints
- recommendations
- simulations
- alternatives
- risks
- confidence
- assumptions

Decision Support exists to improve Executive Work.

---

# Stage 6 — Executive Commitment

Purpose

Convert reasoning into organizational change.

Commitment transforms analysis into active Executive Work.

Discovery persists:

- Executive Decision Records
- expected outcomes
- review schedules
- organizational history

Commitment represents the moment organizational change begins.

---

# Stage 7 — Execution

Purpose

Track active Executive Work.

Discovery should continuously expose:

- execution status
- owners
- expected outcomes
- confidence
- timeline
- progress
- organizational impact

Execution is not static.

The organization continues evolving while work is underway.

---

# Stage 8 — Review

Purpose

Compare organizational expectations with reality.

Discovery evaluates:

```text
Expected Outcomes

↓

Observed Outcomes

↓

Variance

↓

Executive Explanation
```

Executives validate organizational reality through review.

---

# Stage 9 — Learning

Purpose

Improve future executive judgment.

Discovery updates:

- organizational memory
- beliefs
- concepts
- theories
- conditions
- confidence
- recommendations

Learning permanently improves the Operating Model.

---

# Stage 10 — Operating Model Evolution

Purpose

Produce a more accurate understanding of the organization.

Every completed Executive Work item should improve the Operating Model.

Every Operating Model improvement should improve future Executive Work.

Discovery should become progressively better at understanding and improving organizations.

---

# Executive Experience

The interface should expose the organizational lifecycle rather than individual reasoning artifacts.

Executives should naturally experience:

- organizational understanding
- organizational change
- executive work
- execution
- learning

The interface should minimize reading through progressive disclosure.

Every screen should advance the organization through this lifecycle.

---

# Architectural Principles

Every Runtime object belongs to exactly one stage of the Organizational Lifecycle.

Every lifecycle stage produces durable organizational state.

Every durable organizational state contributes to future reasoning.

Every future recommendation should reflect the complete accumulated organizational history.

Discovery succeeds when organizations become progressively better through repeated completion of this lifecycle.
