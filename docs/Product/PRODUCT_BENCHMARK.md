# Product Benchmark

**Status:** Canonical (MVP)

---

# Purpose

This document defines the canonical benchmark used to validate Discovery's MVP.

It answers one question:

> **Can Discovery successfully manage the complete lifecycle of a living organization?**

Unlike architecture documents, this document is **executable**.

Every unchecked item represents missing product functionality.

Every checked item represents a validated end-to-end capability.

No new product capabilities should be added until this benchmark is complete unless they are required to satisfy one of the benchmark stages.

---

# MVP Success Criteria

Discovery is considered MVP complete when an executive can successfully guide a simulated organization through one complete organizational lifecycle and Discovery demonstrably improves its understanding of that organization over time.

The benchmark must be completed from beginning to end without bypassing any stage.

---

# Product Benchmark

## Stage 1 — Organization Creation

### Objective

Create a brand new organization.

### Success Criteria

- [ ] Create organization
- [ ] Organization Runtime created
- [ ] Organization Memory initialized
- [ ] Operating Model initialized

---

## Stage 2 — Organization Onboarding

### Objective

Teach Discovery about the organization.

### Success Criteria

- [ ] Complete onboarding
- [ ] Initial executive context captured
- [ ] Initial organizational objectives captured
- [ ] Initial constraints captured
- [ ] Initial strategic context captured

---

## Stage 3 — Evidence Ingestion

### Objective

Populate the Operating Model with real organizational knowledge.

### Success Criteria

- [ ] Upload board deck
- [ ] Upload strategic plan
- [ ] Upload organizational documentation
- [ ] Upload additional evidence
- [ ] Evidence successfully incorporated
- [ ] Operating Model updated

---

## Stage 4 — Organizational Understanding

### Objective

Verify Discovery understands the organization.

### Success Criteria

- [ ] Organizational Conditions generated
- [ ] Organizational State generated
- [ ] Executive Assessment generated
- [ ] Executive Brief generated
- [ ] Executive can explain why Discovery reached its conclusions

---

## Stage 5 — Executive Work Creation

### Objective

Create work that changes the organization.

### Success Criteria

- [ ] Executive creates external decision
- [ ] Executive creates external initiative
- [ ] Discovery recommends new executive work
- [ ] Executive Work queue populated

Executive Work should become the primary object managed inside Discovery.

---

## Stage 6 — Decision Support

### Objective

### Success Criteria

- [ ] Decision Definition completed
- [ ] Decision Analysis completed
- [ ] Recommendation generated
- [ ] Alternatives evaluated
- [ ] Simulation completed
- [ ] Confidence calibrated
- [ ] Executive explanation available

---

### Success Criteria

- [ ] Executive Decision Record created
- [ ] Executive Work created
- [ ] Expected outcomes generated
- [ ] Review schedule generated

### Objective

Allow the organization to continue evolving.

### Success Criteria

### Success Criteria

- [ ] Executive explanation generated

## Stage 11 — Organizational Learning

### Objective

Improve future executive judgment.

### Success Criteria

- [ ] Learning generated
- [ ] Organizational Memory updated
- [ ] Operating Model updated
- [ ] Confidence recalibrated
- [ ] Recommendations improve

---

## Stage 12 — Continuous Operation

### Objective

Verify Discovery functions as a continuously improving Executive Operating System.

### Success Criteria

- [ ] Morning Brief reflects accumulated organizational history
- [ ] Executive Work reflects accumulated learning
- [ ] Decision tracking persists over time
- [ ] Initiative tracking persists over time
- [ ] Organizational understanding demonstrably improves
- [ ] Future recommendations reflect previous executive decisions
- [ ] Future recommendations reflect observed organizational outcomes

---

# Product Validation Rules

Discovery should never require rebuilding the organization after onboarding.

New evidence should improve the existing Operating Model rather than replacing it.

Every committed Executive Work item should persist throughout its lifecycle.

Every completed review should permanently improve Discovery's understanding of the organization.

The product should continuously evolve with the organization rather than operating as isolated interactions.

---

# MVP Completion

Discovery's MVP is complete when an executive can repeatedly execute this benchmark from beginning to end using a simulated organization and observe Discovery becoming progressively better at understanding and improving that organization through accumulated evidence, executive work, execution, review, and learning.

---

# Development Rule

Until this benchmark passes:

- do not add new reasoning capabilities unless required by the benchmark,
- do not add new product workflows unrelated to the benchmark,
- prioritize completing incomplete benchmark stages over expanding functionality,
- measure progress by completed benchmark stages rather than completed features.

This benchmark is the canonical measure of MVP progress.
